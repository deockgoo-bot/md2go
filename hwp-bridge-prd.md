# PRD: HWP Converter AI Platform
**Product Requirements Document v1.0**
작성일: 2026-03-27 | 상태: Draft

---

## 1. 개요 (Overview)

### 1.1 제품 한 줄 정의
> 공공기관의 HWPX 문서를 AI로 생성·검색·변환·교정하는 문서 자동화 플랫폼

### 1.2 배경 및 문제 정의

| 문제 | 현황 |
|------|------|
| HWP 포맷 종속 | 공공기관 문서의 대부분이 HWPX/HWP — AI 파이프라인 연계 불가 |
| 수작업 공문서 작성 | 기안문·보고서 작성에 평균 1~3시간 소요 |
| 내부 문서 검색 불가 | 파일명 검색만 가능 — 내용 기반 검색 수단 없음 |
| AI 도구 공백 | 기존 AI 도구(ChatGPT 등)는 HWPX 생성·읽기 불가 |

### 1.3 목표
- 공공기관 공문서 작성 시간 70% 단축
- 기관 내부 문서 자연어 검색 실현
- HWPX ↔ Markdown 완전한 양방향 변환 오픈소스 제공
- 2026년 내 PoC 기관 3곳 확보

## 2. 사용자 정의 (Users)

### 2.1 주요 사용자

| 페르소나 | 역할 | 주요 Pain Point |
|----------|------|-----------------|
| 실무 공무원 | 기안문·보고서 작성 | 반복적인 공문서 작성, 유사 문서 찾기 어려움 |
| 기관 IT 담당자 | 시스템 도입·운영 | 망분리 환경, 보안 요건, 온프레미스 설치 필요 |
| 정책 기획자 | 보고서·정책문서 작성 | 방대한 내부 자료 검색, 문서 요약 필요 |
| 개발자 (외부) | API 연동, 오픈소스 기여 | HWPX 파싱/생성 라이브러리 부재 |

### 2.2 사용 시나리오

**시나리오 A — 공문서 초안 생성**
> 김 주무관이 "2026년 AI 도입 예산 기안문"을 작성해야 한다. 플랫폼에 목적과 키워드를 입력하면 기관 공문서 양식에 맞는 HWPX 초안이 2분 내 생성된다.

**시나리오 B — 내부 문서 RAG 검색**
> 박 사무관이 "작년 스마트시티 사업 관련 예산 근거 찾아줘"라고 자연어로 질의하면, 기관 내 HWPX 문서에서 관련 내용과 출처 페이지를 즉시 반환한다.

**시나리오 C — MD → HWPX 자동 출력**
> 개발팀이 Markdown으로 작성한 API 가이드를 공문서 양식 HWPX로 자동 변환해 제출한다.

**시나리오 D — 문서 요약·교정**
> 50페이지 보고서를 업로드하면 1페이지 요약과 행정 문체 교정 결과를 즉시 제공한다.

## 3. 제품 범위 (Scope)

### 3.1 In Scope (MVP)
- HWPX ↔ Markdown 양방향 변환 엔진 (오픈소스)
- 공문서 AI 초안 생성 (기안문·보고서 10종 템플릿)
- HWPX 문서 RAG 인덱싱 및 자연어 검색
- 문서 요약·맞춤법·행정문체 교정
- MD → HWPX 공문서 자동 출력
- REST API
- 웹 UI (파일 업로드/다운로드)
- Docker 온프레미스 패키징

### 3.2 Out of Scope (v1)
- HWP 5.0 바이너리 파싱 (v2에서 검토)
- 한글 97 이하 레거시 포맷
- 모바일 앱
- 전자결재 시스템 직접 연동
- 한↔영 번역 (v2)

## 11. 오픈소스 전략

- **공개 범위**: HWPX ↔ Markdown 변환 엔진 코어 (MIT 라이선스)
- **목적**: 개발자 커뮤니티 바이럴 → 공무원 실무자 입소문 → 기관 도입 건의
- **관리**: GitHub Issues + Discussions 운영, 기여 가이드 작성
- **유료와의 경계**: AI 기능(초안생성·RAG·교정)은 SaaS/온프레미스 유료 전용
